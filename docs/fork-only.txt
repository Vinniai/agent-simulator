# Fork-only vs tracks-upstream boundary manifest (ADR-0001)
#
# agent-sim is a selective clone of tddworks/baguette. Code that maps cleanly to
# upstream ("tracks-upstream") must stay upstream-shaped so commits cherry-pick
# with only a path remap; code that exists only to serve the Agentic Feedback
# Loop ("fork-only") may evolve freely because it costs nothing at absorb time.
#
# This file is the single source of truth for `scripts/check-fork-boundary.sh`,
# which enforces two rules:
#   1. completeness — every context directory under Domain/ and Infrastructure/,
#      and every file under App/Commands/ and App/*.swift, must be classified in
#      exactly one [classify:*] section below. A new, unclassified top-level
#      source file fails the build (forces a conscious zone decision).
#   2. leakage — no file inside a tracks-upstream context may reference a
#      fork-only type named in [markers]. That coupling would drag loop value
#      into the upstream-shared zone and break the next cherry-pick.
#
# Categories:
#   tracks-upstream — keep close to upstream; cherry-picks must stay cheap.
#   fork-only       — loop value; evolve freely.
#   diverged        — accepted manual-port divergence (re-architected vs upstream).
#   shared          — wiring both zones legitimately touch (exempt as a leak source).
#
# Format: `[classify:<layer>] <category>` header, then one bare name per line
# (a directory name for Domain/Infrastructure layers, a filename for app layers).
